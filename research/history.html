開発メモ / Evolution Notes

ここまでどう作ってきたかを、自分用メモとして整理する。

このページは、Kendo Virtual Coachを作りながら考えたこと、変更したこと、やめたことを残すためのメモです。 最初は「剣道の動きを数値で見たい」というところから始まり、途中でYOLO版も試しつつ、最終的にはMediaPipe版と動画解析を中心にする形へ寄せました。 卒研で説明するときに、「なぜ今の仕様になったのか」を振り返れるようにまとめています。

01 / CURRENT STATE

現在の到達点

現在は、MediaPipe版を本採用として進めている。 リアルタイム分析も残しているが、打突時間や角度をきちんと確認する場面では、動画解析を中心に使う方針にした。 理由は、動画なら開始フレームと終了フレームを自分で確認でき、スローで見直しながら測定値を説明しやすいからである。

本採用MediaPipe Pose Landmarker中心
正式測定高fps動画解析 + 手動補正
主指標打突時間・ピーク時刻・角度
研究説明卒研情報センターに分離
02 / TIMELINE

開発フェーズ

下の履歴は、単に追加した機能を並べるのではなく、その時に何を考えて、なぜ今の方向へ変えたのかを残したものである。 卒研で説明するときは、この流れを見せると「行き当たりばったりではなく、目的に合わせて絞ってきた」と説明しやすい。

Phase 1初期構想

剣道の動きを数値で見たいところから開始

最初は、剣道の打突動作をカメラで見て、良い動きかどうかを数値で確認できるツールを作ろうとした。

  • まずは打突動作を定量評価する方向で作り始めた。
  • カメラ映像の上に骨格や評価を重ねる形にした。
  • 最初はGOOD/CHECKのような分かりやすい表示も入れていた。
定量評価リアルタイム単体HTML
Phase 2アプリ化

スマホやPCで使えるようにアプリ化

最初のHTMLだけでは使いにくかったため、スマホに追加できる形や、localhostで安定して開く方法を整えた。

  • manifest.webmanifestとsw.jsを追加した。
  • file直開きではカメラやAIが不安定なので、localhostで開く運用にした。
  • スマホやPCで試しやすいように、起動方法も整理した。
PWAlocalhost起動安定化
Phase 3MediaPipe基盤

MediaPipeで上半身の角度を見る方向へ

剣道の打突動作では、まず上半身の動きが重要だと考え、肩・肘・手首・腰の座標から角度を計算する形へ進めた。

  • 右肘角、体幹軸、刃筋を出せるようにした。
  • 面・小手・胴・突きで評価基準を変える案も検討した。
  • ただし最初の卒研では、足元よりも上半身・ピーク時刻・角度に絞る方が良いと判断した。
MediaPipe角度上半身特化
Phase 4稽古UI

練習画面をシンプルにする方向へ

作っていくうちに画面の情報が増えすぎて、実際の練習では見にくいと感じた。そのため、練習中に見る情報を減らす方向にした。

  • 練習画面を最大化できるようにした。
  • 打突時間、角度、参考ピークを中心に表示するようにした。
  • GOOD/CHECKが細かく切り替わると見にくいため、表示の安定性を重視するようにした。
UX改善不要項目整理稽古用
Phase 5YOLO検証

YOLO版も試したが、比較版として残すことにした

MediaPipeより精度を上げられる可能性を考えて、YOLO Pose版も作った。ただ、ブラウザ単体で安定して使うには準備や動作の重さが課題になった。

  • YOLO版は消さずに、比較・将来検証用として残した。
  • 上半身の速度や角度を取る方向性は試した。
  • 卒研の本採用としては、現時点で説明しやすいMediaPipe版に戻した。
比較版YOLO PoseONNX Runtime Web
Phase 6動画解析

動画解析を正式測定の中心にした

リアルタイム分析は便利だが、打突時間を正確に見るには限界がある。そこで、iPhoneなどで撮った120fps/240fps動画を後から解析する方向に重点を移した。

  • 動画読み込み、スロー再生、フレーム送りを追加した。
  • 開始/終了フレームを自分で置けるようにした。
  • 動画フレームとピーク・角度グラフを合わせて見られるようにした。
高fps手動補正正式測定
Phase 7信頼性

測定値をそのまま信じない仕組みを追加

数字が出ても、それが正しいとは限らない。そのため、短すぎる打突時間、速度の急な跳ね、骨格の飛び、信頼度の低下などを要確認として表示するようにした。

  • 妥当性チェックを一覧で見られるようにした。
  • 異常値除外の基準を作れるようにした。
  • 測定値を採用してよいか説明しやすくした。
異常値除外信頼度卒研向け
Phase 8卒研分離

卒研説明は別ページに分けた

解析アプリの中に研究説明まで入れると、練習中の画面が複雑になる。そのため、研究説明は卒研情報センターとして分けた。

  • /research/ に卒研情報センターを作った。
  • 検索、メニュー分類、FAQ、用語集を入れた。
  • 研究課題、関連技術、実験プロトコル、卒論章立ても整理した。
研究説明検索別サイト化
03 / DECISION LOG

主な仕様判断

開発中に迷った点と、最終的にどう判断したかをメモとして残す。

MediaPipeを本採用ブラウザで動かしやすく、動画解析とリアルタイム分析を両方残しやすいため。
YOLOは比較版として残す精度面の可能性はあるが、今の段階では安定運用まで持っていくのが難しいため。
動画解析を重視打突時間は一瞬なので、リアルタイムより動画で確認した方が説明しやすいため。
足元系は後回し最初から足元まで入れると研究範囲が広くなりすぎるため。
卒研説明は別サイト練習画面をシンプルに保ちつつ、研究説明だけ詳しくできるため。
測定器とは言い切らない速度や角度は撮影条件に左右されるため、比較・振り返り用として扱うため。
04 / FEATURE MEMO

これまでに入れた主な機能

ここは、今のアプリに何が入っているかを自分で確認するためのメモである。 機能を増やした目的は、ただ多機能にすることではなく、打突時間・ピーク時刻・角度を説明できる形に近づけることである。

分類入れたものなぜ入れたか
測定打突時間、最高速時刻、右肘角、体幹軸、刃筋剣道の素振りを見たときに、まず確認したい数字だから。
動画解析動画読み込み、手動開始/終了、スロー再生、フレーム補正リアルタイムだけでは怪しい部分を、あとから1コマずつ確認できるようにするため。
信頼性姿勢信頼度、骨格飛び、速度異常、時間異常のチェック数字が出ても、明らかにおかしい値はそのまま使わないようにするため。
可視化骨格、角度ライン、速度/角度グラフ、3Dモデルどの関節や線を基準にしているかを、見た目で説明できるようにするため。
記録履歴、CSV、レポート、正式記録保存1本だけで判断せず、複数回の動きを比べられるようにするため。
研究説明卒研情報センター、検索、FAQ、用語集、開発メモアプリを使う画面と、卒研として説明する画面を分けるため。
05 / DEFERRED ITEMS

いったん保留・削除した機能

作っている途中で、面白そうだから入れたくなった機能も多かった。 ただ、最初の卒研では打突時間・ピーク時刻・角度をできるだけ説明できる形にすることが大事なので、目的から外れるものはいったん後回しにした。

気剣体一致・気合音量剣道では大事だが、音声や主観評価が入るため、今回の測定テーマからは外した。
足幅・歩幅・踏み込み詳細足元まで入れると研究範囲が広がりすぎる。まずは上半身と打突動作に絞る。
複雑なGOOD/CHECK表示表示がパカパカすると練習中に見づらいので、判定よりも数値と安定表示を優先する。
過度にリッチな3D演出かっこよさより、角度の基準が分かることを優先する。3Dは説明用として使いやすくする。
YouTube直接解析ブラウザの制限や権利の問題があるため、基本は自分で用意した動画ファイルを読み込む形にする。
市販測定器相当の絶対速度現段階では推定値として扱う。同じ条件で比較することを重視する。
06 / PRESENTATION STORY

先生に説明するときの流れ

説明するときは、最初から完成形を作ったというより、使いながら必要な形に絞っていった、と話すのが自然だと思う。 最初はリアルタイムでGOOD/CHECKを出す練習支援ツールとして作り始めた。 しかし、実際に使ってみると、打突時間はかなり短く、リアルタイム処理だけではフレーム落ちやノイズの影響を受けやすいことが分かった。

そのため、練習中はリアルタイムで大まかに確認し、正式な測定は高fps動画を読み込んで、開始フレームと終了フレームを手動で確認する形に変えた。 さらに、測定値が出るだけではなく、その値を採用してよいか判断するために、信頼度や異常値チェックも入れた。 最終的に、練習者が使う解析アプリと、卒研として説明する情報ページを分ける形にした。

発表では「機能をたくさん増やした」ではなく、「測定値を説明できるようにするため、リアルタイム中心から動画解析中心へ整理した」と言うと伝わりやすい。
